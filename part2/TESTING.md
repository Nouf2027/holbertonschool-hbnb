# Manual Testing Report – Task 6 (HBnB Part 2)

This document describes the **manual black-box testing** performed for Task 6
using `cURL` and the Swagger documentation generated by Flask-RESTx.

The goal is to confirm that the API endpoints:
- Enforce validation rules correctly
- Handle invalid input gracefully
- Return correct HTTP status codes

---

## Environment

- Base URL: http://127.0.0.1:5000
- API Prefix: /api/v1
- Swagger UI: http://127.0.0.1:5000/api/v1/
- Framework: Flask + Flask-RESTx
- Storage: In-memory repository

---

## 0. Swagger (Documentation) Check

### Swagger UI Available

Command:
curl -i http://127.0.0.1:5000/api/v1/

Expected Result:
- Status Code: 200 OK

Result:
- PASS

## 1. User Endpoints

### Create User (Valid)

Command:
curl -X POST http://127.0.0.1:5000/api/v1/users/ \
-H "Content-Type: application/json" \
-d '{
  "first_name": "John",
  "last_name": "Doe",
  "email": "john.doe@example.com"
}'

Expected Result:
- Status Code: 201 Created

Result:
- PASS

---

### Create User (Invalid Email)

Command:
curl -X POST http://127.0.0.1:5000/api/v1/users/ \
-H "Content-Type: application/json" \
-d '{
  "first_name": "John",
  "last_name": "Doe",
  "email": "invalid"
}'

Expected Result:
- Status Code: 400 Bad Request

Result:
- PASS
