# HBnB API – Manual Testing Documentation

This document describes the manual (black-box) testing process for the HBnB API.
All tests were executed using `curl` commands and verified through the Swagger UI
generated by Flask-RESTx.

---

## Prerequisites

- Flask application running locally
- Base URL: http://127.0.0.1:5000
- Swagger URL: http://127.0.0.1:5000/api/v1/
- `curl` installed

---

## Tested Endpoints Overview

The following entity endpoints were manually tested:

1. Users
2. Amenities
3. Places
4. Reviews

---

## 1. Users Endpoint

### Create User – Valid Input

```bash
curl -X POST http://127.0.0.1:5000/api/v1/users/ \
-H "Content-Type: application/json" \
-d '{
  "first_name": "John",
  "last_name": "Doe",
  "email": "john.doe@example.com"
}'
```

Expected Result:
- Status Code: 201 Created
- User object returned with generated ID

---

### Create User – Invalid Email

```bash
curl -X POST http://127.0.0.1:5000/api/v1/users/ \
-H "Content-Type: application/json" \
-d '{
  "first_name": "John",
  "last_name": "Doe",
  "email": "invalid"
}'
```

Expected Result:
- Status Code: 400 Bad Request

---

### Create User – Empty First Name

```bash
curl -X POST http://127.0.0.1:5000/api/v1/users/ \
-H "Content-Type: application/json" \
-d '{
  "first_name": "",
  "last_name": "Doe",
  "email": "x@y.com"
}'
```

Expected Result:
- Status Code: 400 Bad Request

---

### Get User – Not Found

```bash
curl http://127.0.0.1:5000/api/v1/users/does-not-exist
```

Expected Result:
- Status Code: 404 Not Found
